pg_result_error

(PHP 4 >= 4.2.0, PHP 5, PHP 7, PHP 8)

pg_result_error — 获得跟 result 相关的错误信息



说明

pg_result_error(PgSql\Result $result): string|false

pg_result_error() 返回和 result 实例关联的错误信息。因此用户更有机会可以得到比 {@link pg_last_error()} 更好的错误信息。

函数 {@link pg_result_error_field()} 可以比 pg_result_error() 给出更多关于 result 错误的细节。

因为如果查询失败 {@link pg_query()} 返回 false，必须使用 {@link pg_send_query()} 和 {@link pg_get_result()} 来获取 result 句柄。



参数

result

PgSql\Result 实例，由{@link pg_query()}、{@link pg_query_params()} 或者 {@link pg_execute()}（等）返回。


返回值

返回 string。如果没有错误返回空字符串。如果有跟 result 参数相关的错误，则返回 false。



更新日志

版本 说明
8.1.0 现在 result 参数接受 PgSql\Result 实例，之前接受 resource。


示例

示例 #1 pg_result_error() 示例

<?php
$dbconn = pg_connect("dbname=publisher") or die("Could not connect");

if (!pg_connection_busy($dbconn)) {
pg_send_query($dbconn, "select * from doesnotexist;");
}

$res1 = pg_get_result($dbconn);
echo pg_result_error($res1);
?>


参见

  • {@link pg_result_error_field()} - Returns an individual field of an error report
  • {@link pg_query()} - 执行查询
  • {@link pg_send_query()} - 发送异步查询
  • {@link pg_get_result()} - 取得异步查询结果
  • {@link pg_last_error()} - 得到某连接的最后一条错误信息
  • {@link pg_last_notice()} - 返回 PostgreSQL 服务器最新一条公告信息
