db2_stmt_error

(PECL ibm_db2 >= 1.0.0)

db2_stmt_error — Returns a string containing the SQLSTATE returned by an SQL statement



说明

db2_stmt_error(?resource $stmt = null): string

Returns a string containing the SQLSTATE value returned by an SQL statement.

If you do not pass a statement resource as an argument to db2_stmt_error(), the driver returns the SQLSTATE value associated with the last attempt to return a statement resource, for example, from {@link db2_prepare()} or {@link db2_exec()}.

To learn what the SQLSTATE value means, you can issue the following command at a DB2 Command Line Processor prompt: db2 '? sqlstate-value'. You can also call {@link db2_stmt_errormsg()} to retrieve an explicit error message and the associated SQLCODE value.



参数

stmt

A valid statement resource.



返回值

Returns a string containing an SQLSTATE value.



参见

  • {@link db2_conn_error()} - Returns a string containing the SQLSTATE returned by the last connection attempt
  • {@link db2_conn_errormsg()} - Returns the last connection error message and SQLCODE value
  • {@link db2_stmt_errormsg()} - Returns a string containing the last SQL statement error message
